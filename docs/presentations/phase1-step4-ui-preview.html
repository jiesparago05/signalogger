From a signal meter
to a connectivity map

Step 4 introduces a 4-state classification into the map UI, replacing the old dBm-only gradient. It also surfaces sample count (so users can trust crowd data), freshness (how recent a reading is), and differentiates the dead-zone banner between "no signal" and "no internet" — the two most confusing cases a user can hit.

The four states

Every dot means something now

Instead of a single dBm gradient (green → red), every reading classifies into one of four states. The classification uses both signal strength and Android's own NET_CAPABILITY_VALIDATED probe — which is how the app distinguishes "weak signal but internet works" from "strong signal but data is suspended."

OK
Strong signal AND Android confirms internet is working.
// classifyZone()
dbm > -95 &&
validated === true
No alert
WEAK
Working but degraded. Ambiguous readings land here — e.g., signal OK but Android hasn't probed yet.
// everything in between
dbm > -115 &&
validated !== false
No alert
NO INTERNET
Signal is fine, but Android confirmed the internet is broken. Usually means data plan exhausted or backhaul failure.
// explicit false only
validated === false
Fires alert (new)
NO SIGNAL
Radio off, native read failed, or signal below the dead-zone threshold. No data possible.
dbm <= -115 ||
dbm === -999 ||
networkType === 'none'
Fires alert

Confidence and freshness

When a user opens the Signal Summary for a consolidated dot, they now see how many readings the data is based on and when the most recent one was captured. Without these, crowd-sourced data is hard to trust — a single stale reading and a fresh 100-sample average look identical.

What's new in the summary

✔ Classification state replaces the raw dBm-only header. "OK" instead of just "-74 dBm avg."

✔ Sample count badge — "Based on 47 readings" — lets a user know whether they should trust the data.

✔ Validated ratio — when consolidated data has the new fields, show "46/47 validated" so users can see whether most of those readings had internet.

✔ Freshness dot — green when < 1 hr, gray when old. "Updated 12 min ago."

✔ Falls back gracefully when sample count is 1 or when old readings don't have the new fields — shows "Single reading" or omits validated entirely.

Dead zone banner

Two reasons, two messages

The existing banner fires on one condition: dBm below threshold. Step 4 differentiates the root cause. "Dead Zone" and "No Internet — Signal OK" look different and say different things because they require different user action.

NO_SIGNAL (existing behavior)
⚠️
Dead Zone — No Signal
Data will sync when signal returns
// fires when...
dbm <= -115 || dbm === -999
// OR networkType === 'none'
NO_INTERNET (new)
⚠️
No Internet — Signal OK
Check your data plan or network status
// fires when...
validated === false
// for 3 consecutive readings

Why two banners?

The user actions are completely different:

NO SIGNAL  → wait, move to higher ground, try the other SIM. Nothing works.

NO INTERNET  → check load balance, restart mobile data, check roaming. Signal bars are present — the problem is elsewhere.
